What this tool does

A reusable audit workspace for CleverTap retention journeys. Drop in your journey CSV exports — for any product, any category, any time period — and the dashboard rebuilds itself from the data. It only shows what's actually in your files: no auto-generated recommendations, no fabricated insights.

01

What you can put in

Any CleverTap journey export CSV — one file or many. The tool dedupes across files automatically (handles the iOS/Android row-split that CleverTap exports for some journeys). Drop them anywhere on the page, or click "Load CSV files" above.

02

What you get out

Per-journey deep-dives (every message, full copy, all the metrics), topline aggregates, a searchable inventory, and auto-generated Data Quality flags pointing at things like duplicate journey names, running-but-silent journeys, and push-fatigue risk.

03

What it doesn't do

It doesn't write recommendations for you. Auto-generated insights are limited to mechanical rules over your data (e.g. "these journeys show as Running but have zero sends"). Bespoke strategic recommendations are a separate exercise per product.

04

How sharing works

This is a static page. Email the bundled HTML to anyone — they can open it in any browser and load their own CSVs. Loaded data stays in their browser only (via local storage); nothing is uploaded anywhere.

How to use it — three steps

  1. Open this file in any browser. Right now you're looking at the Hair / Stage 2 sample data so the tool isn't empty. The orange "DEMO" badge in the bar above marks this. No installation required. Works in Chrome, Safari, Firefox, Edge.
  2. Drop your CleverTap CSV files anywhere on the page — or click "+ Load CSV files" in the bar above. Drop multiple files at once; the tool will dedupe across them. The DEMO badge flips to UPLOADED, and the dashboard rebuilds. Expected columns: Journey ID, Journey Name, Version Status, Node ID, Node Type, Total Sent, Total Delivered, Total Viewed, Total Clicked, Unique Converted within Conversion Time, Influenced Conversions, Influenced Revenue, Goal 1/2/3 Name + Conversions, Campaign Message, Campaign / Segment / Controller Name + Type, Campaign OS, Delay before Node, Connector from Previous. (Standard CleverTap export schema — should match out of the box.)
  3. Browse the tabs. Overview for the topline, Journeys for the full searchable list with deep-dives, Data Quality for auto-flags, Methodology for what each metric means. Your loaded data persists in your browser's local storage, so you don't have to re-upload every time. Click "Reset to demo" to clear and go back to the sample data.

What this tool does NOT replace

For incremental conversion measurement (true journey lift), you still need Control Group data from CleverTap's Campaign Report — it isn't in the standard journey export schema. For bespoke strategic recommendations (a consolidated journey design for a specific product, or a copy-rotation proposal), this tool surfaces the inputs but the synthesis is a human exercise. Frequency-per-customer caps need to be cross-checked in CleverTap's Global Campaign Limits settings — not exported here.

How the tool processes your data

All processing happens in your browser. Nothing is sent to a server.

Dedup logic

CleverTap journey exports have two kinds of duplication:

  • Within-file: rows split by Campaign OS (Android, iOS). The same node fired to both OSs appears as two rows — one per OS. If not collapsed, you lose one OS's metrics.
  • Cross-file: if you export the same journey in multiple files (e.g. as part of a "category" export and a "specific journey" export), each file will have a complete copy of that journey's rows.

The tool handles both:

  1. For each Journey ID, pick one source file — the first file (in the order you loaded them) that contains that journey.
  2. Within that file, group by (Journey ID, Node ID) and sum the metric columns. This collapses OS-split rows into one row per node, with metrics correctly added across OSs.
  3. The result is one row per (Journey, Node) with correctly-summed lifetime metrics.

If a journey shows up in two files with DIFFERENT row counts or DIFFERENT metric values, the version from the first-loaded file wins. The other is silently dropped. (If you need different precedence, reorder your file uploads.)

What each metric means

  • Sent — for message nodes, attempts to send. For webhook nodes, trigger fires (no comm delivered).
  • Delivered — populated for WhatsApp and SMS. NOT populated for Push (CleverTap doesn't track push delivery the same way).
  • Viewed — Push impressions + WhatsApp read receipts.
  • Clicked — push notification taps. Not tracked here for WhatsApp / SMS (those use link tracking off-platform).
  • Converted — unique customers who fired the conversion event (typically "Charged") within the configured conversion window. Direct attribution.
  • Influenced Conversions / Revenue — broader attribution including post-window and assist conversions. Usually the headline number.
  • Goal 1/2/3 Conversions — counts of customers who hit the configured Goal event for each goal slot. "My segment" goals are funnel-step exits, not outcomes — interpret with care.

Quality flag rules

The Data Quality tab applies these rules to your data:

  • Silent Running journeys — Status = Running but Total Sent = 0 across all message nodes. Likely cause: entry condition is broken or never matches.
  • Duplicate journey names — two or more Journey IDs share the same name. Often a sign of a deprecated version that should be archived.
  • Push-heavy journeys — 10+ Push nodes vs ≤ 1 WhatsApp node. Push impressions are usually healthy but CVR-per-push is far lower than CVR-per-WhatsApp; fatigue risk if Global Campaign Limits aren't tight.
  • Webhook-only journeys — every node is a Webhook. The "sent" count counts trigger fires, which is misleading if rolled up into comm-volume KPIs.
  • WA/SMS without delivery data — WhatsApp or SMS nodes where Total Sent > 0 but Total Delivered = 0. Tracking gap to look into.
  • No Goal events configured — journeys with no Goal 1/2/3 Name populated. Direct conversion attribution can't be measured.
  • Paused journeys still in the workspace — Status = Paused. Listed for archival review.

Rules are defined in flags.js. Add or edit rules there.

What this tool does NOT compute

  • Control-Group / incremental lift — Control Group counts aren't in the journey export schema. Get from the Campaign Report subscribe flow separately.
  • Frequency per customer — these are journey-level aggregates, not customer-level.
  • A/B winner determination — IntelliNODE winner data isn't always present in the export. Where it is, it'll show on the per-node view.
  • Year on dates — CleverTap date fields don't include the year. The tool shows the raw date string from your export.

Source code & privacy

This is a small static web app. The only third-party library is PapaParse for CSV parsing (MIT license, ~19KB). Your uploaded data never leaves your browser — it's processed locally and (optionally) saved to local storage on your machine for persistence across page reloads. Hitting "Reset to demo" clears it.
